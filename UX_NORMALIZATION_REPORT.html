Overview · 01
UX Normalization
19 normalized interaction patterns that all 7 products must follow. Each rule is anchored to the best existing example inside Direct. “Apply to” scope per vertical will be filled in separately.
01
Back Button
Top-left, 44×44px hit area. Use destination label on key flows (“Back to Search”), not a bare chevron. On data-loss flows, trigger a confirmation sheet before navigating away.
Best practice: Visa Requirements — back button with flow-labeled breadcrumb
02
Bottom Sheet
Drag handle + × close button (44×44px) required on every sheet. Drag-only dismiss fails WCAG 2.5.3. Sheet height 50–90% of screen. Must not clip on short-display devices.
Best practice: Figma passenger picker spec — × top-left in RTL, drag handle retained
03
Page Title
Title reflects the current page context, not the brand name. Sticky on scroll. Cairo 700, 18px. One line max — truncate with ellipsis if longer.
Best practice: Visa Requirements — “متطلبات التأشيرة” as header title
04
Subtitle
Secondary line below Page Title. Cairo 400, 13px, muted color. Used for route, traveller count, or date range (e.g. القاهرة → الرياض · 3 ليالٍ).
Best practice: Pending — scope to be defined per vertical
05
Search Results List
Row min-height 64px. Leading icon/flag + title + subtitle layout. Trailing chevron direction-aware in RTL. Selection highlights row with var(--gold) tint.
Best practice: Flights airport picker — IATA + city + country, instant filter, gold row on tap
06
Filter Modal / Sheet
Always a bottom sheet. Active filter count badge on trigger button. Clear-all and Apply CTAs fixed at sheet bottom, never scrolled away.
Best practice: Hotels filter sheet — active count badge, clear all, Apply pinned to bottom
07
Sort Options
Bottom sheet or compact action sheet. Active option: filled radio + var(--gold). Default sort labeled explicitly. Max 5 options to avoid choice paralysis.
Best practice: Flights sort sheet — labeled default, gold active state
08
Primary Button
Full-width, 56px height, var(--gold) fill. Cairo 700, 16px. Copy: [Verb] + [Object] max 3 words. Anchored to bottom safe area with safe-area-inset-bottom padding.
Best practice: Visa — “ابدأ طلب التأشيرة”
09
Secondary Button
Outlined: 56px height, var(--gold) border + text, transparent fill. Stacks directly above Primary. Position never swaps. Ghost button for tertiary actions.
Best practice: Pending — scope to be defined per vertical
10
Loading State
Button enters loading with inline spinner + present-continuous Arabic label (“جاري الحجز…”). Full color retained; opacity unchanged. Tap blocked during load.
Best practice: Booking confirmation flow — spinner inside button + Arabic label
11
Card Container
12px border-radius, 1px border (var(--bd)), 16px padding. Shadow: 0 2px 8px rgba(0,0,0,.06). Image aspect ratio 16:9 or 4:3. Text must never be clipped.
Best practice: Hotels search card — consistent radius, shadow, and image ratio
12
Badge (Sale / New)
Overlays card top-left in LTR, top-right in RTL. Max 2 badges per card. Sale → var(--red), New → var(--blue), Featured → var(--gold). Cairo 700, 10px.
Best practice: Pending — scope to be defined per vertical
13
Empty States
Auth-aware: unauthenticated → login CTA; no results → illustration + retry CTA; post-filter empty → “clear filters” CTA. A blank screen is never acceptable.
Best practice: My Orders — illustration + Arabic label. Gap: no login CTA when unauthenticated
14
Skeleton Screens
Skeleton shimmer cards matching result card shape for any load > 500ms. Reserve spinner for sub-500ms inline operations only.
Best practice: Figma flights loading — “جاري البحث عن رحلات” + skeleton cards
15
Layout Mirroring
RTL flips all directional elements: chevrons, shadows, padding offsets, slide animations. Navigation reads right-to-left. Always test with the longest Arabic string.
Best practice: Correct in Flights, Hotels, Visa. Failure: Packages browse cards use wrong chevron direction
16
Arabic Font
Cairo for all Arabic text. Weights: 400 body, 700 labels, 900 headings. Min 13px body, 11px captions. Never fall back to system default Arabic font.
Best practice: Visa Requirements — Cairo consistently applied across all text sizes and weights
17
Numerals / Calendar
Eastern Arabic numerals (٠١٢٣٤٥٦٧٨٩) in Arabic locale. Calendar: Gregorian default, Hijri as secondary label. Date format: DD/MM/YYYY.
Best practice: Pending — scope to be defined per vertical
18
Transition
Screen push: 280ms ease-out slide. Modal/sheet entry: 320ms spring (cubic-bezier). No abrupt cuts on primary navigation. Must respect OS reduce-motion setting.
Best practice: Figma motion spec — spring curves for sheets, ease-out slide for screen transitions
19
Loading
Full-page loading: centered spinner + Arabic label (“جاري التحميل…”). Overlay loading (e.g. payment): semi-transparent dark backdrop + centered spinner.
Best practice: Pending — scope to be defined per vertical
