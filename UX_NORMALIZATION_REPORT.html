Overview · 01
UX Standards
These are the interaction patterns that must apply across the entire app — all 7 products. Each standard is anchored to the best existing example inside Direct. Where no good example exists, the Figma spec or industry best practice is the reference.
01
Search Experience
Every destination/location input must open a full bottom sheet with a live-filter search bar, results list with flag or icon, and row highlight on selection. Minimum 44px row height.
Best practice in app: Flights airport picker — IATA code + city name + country, instant filter, orange row on tap
Apply to
Hotels destination Visa country Study city eSIM country
02
Primary CTA — Verb + Object
Every main action button must follow the formula [فعل أمر] + [الموضوع], max 3 words. Full-width (padding 16px), 56px height, anchored to the bottom safe area. Background: var(--gold).
Best practice in app: Visa Requirements — "ابدأ طلب التأشيرة"
Apply to
Flights → ابحث عن رحلات Hotels → ابحث عن فنادق Packages → اختر هذه الباقة Study → ابدأ طلب القبول eSIM → احصل على الشريحة License → استخرج رخصتي
03
Bottom Sheet — Close Button
Every bottom sheet must have a visible × close button (top-right, 44×44px hit area) in addition to the drag handle. Drag-only dismiss fails WCAG 2.5.3 for motor-impaired users.
Best practice: Figma passenger picker spec — × in top-left (RTL), drag handle retained
Apply to
All bottom sheets Airport picker Country picker Date picker Passenger picker
04
Header Title — Page Context, Not Brand
The header title must reflect the current page or destination name, not the app brand "Direct". Brand is already conveyed by icon, splash screen, and app store presence.
Best practice in app: Visa Requirements — "متطلبات التأشيرة" as header title
Apply to
Flights header Hotels header Packages header All drill-downs
05
Date Selection — Range Picker
For any round-trip or multi-day booking, departure and return dates must be selectable in a single calendar interaction. A two-step date flow adds unnecessary friction.
Best practice: Figma date range picker — الذهاب + العودة in one sheet, Arabic month grid
Apply to
Flights round-trip Hotels check-in/out Packages dates
06
Passenger & Class Selector
Passenger count uses stepper (+/−) with type labels (بالغ, طفل, رضيع) and age ranges. Class uses a radio list. Both live in a single bottom sheet with "تأكيد" CTA.
Best practice: Figma passenger picker spec — grouped sections, × close, clean white surface
Apply to
Flights Hotels (guests + rooms) Packages
07
Trip Type Segment Control
Screens with multiple trip variants must use a pill segment control at the top of the form. Active pill: white on var(--gold). Height: 40px. Font: Cairo 14px medium.
Best practice in app: Flights — ذهاب / ذهاب وعودة / رحلات متعددة
Apply to
Hotels (1 night / multi) Packages (type) Visa (single / multiple)
08
Empty State — Contextual, Not Generic
Empty states must detect context: unauthenticated users see a login prompt; no-results states show an illustration + retry CTA; post-search empty shows "no flights found" + suggest alternatives.
Best practice: My Orders — illustration + Arabic label. Gap: no login CTA when unauthenticated.
Apply to
My Orders (auth gate) Search results (no match) Visa tab (after select)
09
RTL — Chevron Direction
In Arabic RTL layout, forward navigation chevrons must point › (right-pointing). A ‹ chevron means "go back" in RTL reading direction. All card and list row chevrons must be direction-aware.
Correct in app: Flights, Hotels, Visa form fields. Failure: Packages browse cards use ‹
Fix in
Packages cards More tab cards Any new card component
10
Loading State — Skeleton Screen
Search result loading must show a skeleton screen (grey placeholder cards matching the result card shape), not a spinner. The Figma spec shows the correct skeleton pattern for flights.
Best practice: Figma flights loading screen — "جاري البحث عن رحلات" + skeleton cards
Apply to
Flights results Hotels results Study Abroad results All search flows
